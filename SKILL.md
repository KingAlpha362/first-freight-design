Read the `README.md` file within this skill, and explore the other available files
(`colors_and_type.css` for tokens, `assets/` for the logo and brand photography,
`ui_kits/website/` for ready-made marketing-site components, `preview/` for visual specimens).

If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and
create static HTML files for the user to view. If working on production code, copy assets and read
the rules here to become an expert in designing with the First Freight brand.

Core rules of thumb:
- One hot **red-orange (`#DE4632`)** is the entire identity — use it as a punch (CTAs, the
  wordmark, key numbers), never as a large wash. Hi-vis **orange (`#F26A21`)** is the accent.
- Render the wordmark from `assets/logo-first-freight.png` — never re-type it.
- Display type = Zilla Slab (often italic for brand moments); UI/body = Barlow; labels, fleet
  codes & tracking numbers = Barlow Semi Condensed. (These are Google-Font substitutes — see README.)
- Real, candid operations **photography** for backgrounds, with a dark bottom-up protection
  gradient behind any text. No illustration, no abstract gradients, no emoji.
- Icons: Lucide (2px stroke), loaded from CDN.
- Voice: plain-spoken, dependable, "your / our". Tagline: *"Your Parcel, Our Priority."*

If the user invokes this skill without any other guidance, ask them what they want to build or
design, ask a few clarifying questions, and act as an expert designer who outputs HTML artifacts
_or_ production code, depending on the need.
